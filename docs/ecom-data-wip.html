Methodology

Data Sources

5 channels tracked weekly:

  • Olympikus — olympikus.com.br (official website)
  • Mizuno — mizuno.com.br (official website)
  • Centauro — centauro.com.br (marketplace)
  • Direct — adidas.com.br + nike.com.br + asics.com.br + underarmour.com.br
  • Netshoes — netshoes.com.br

BQ tables: olympikus_trusted, mizuno_trusted, centauro_trusted, usr_andre_adidas_nike_product_snapshot_2026_05_15, usr_andre_netshoes_product_snapshot_2026_05_15

Data Grain

  • Direct / Netshoes: colorway level (model + color) — confirmed: Adidas, Nike, Asics and Under Armour do not vary price by size; all sizes of the same colorway share one price
  • Olympikus / Mizuno: EAN level (model + color + size) via Aster pipelines
  • Centauro: EAN level after seller deduplication — see below

% SKUs with Discount

COUNTIF(sale_price < list_price) / COUNT(*) at model level, by week and category.

A model is "discounted" when its sale price is strictly below its listed price.

Average Discount Depth

Simple mean of (1 − sale/list) across discounted models per week & category.

When aggregating categories, weighted average by n_disc (number of discounted models).

Sport Category Filter

Each channel uses its own naming convention, mapped to three buckets:

  • Corrida — running only (Corrida / Running / Corrida·Caminhada)
  • Performance — Corrida + Treino + Academia/Fitness + Trilha (channel-specific name variations)
  • All — all footwear categories

Raw category names differ by source: Direct uses VTEX/API sport names (Running, SportStyle, Treino & Academia…); Netshoes uses its own taxonomy (Corrida, Treino, Trail Running, Fitness e Musculação…). Each is mapped individually in SPORT_CATS.

Centauro Marketplace

The same product (EAN = model + size) can be listed by multiple third-party sellers at different prices.

Methodology: for each EAN on each date, keep only the lowest available price.

Rationale: measures the discount level present for each brand in the market — if a shoe is available at a discount anywhere on the site, that product is discounted.

Week Convention

  • ISO week labelled by its Sunday — e.g. W20 2026 = 2026-05-17
  • Olympikus / Mizuno (Aster): daily collection — week aggregates all 7 days
  • Direct (Adidas/Nike/Asics/UA): daily scraper 07:00 BRT — week aggregates all daily snapshots collected Mon–Sun
  • Netshoes (own scraper): daily collection — week aggregates all available days
  • Centauro (Aster): Thursday snapshot only — 1×/week. The "week" datapoint for Centauro is in fact a single Thursday snapshot. When that Thursday is missed by the source, the entire week is empty (no fallback to other days).

Centauro data gaps: (a) structural — Black Friday and any weekend event are never captured since collection is Thursday-only; (b) historical — no Centauro data exists in the source between 2023-10-01 and 2025-07-27 (~21 months); (c) sporadic missed Thursdays: 2025-09-18, 2025-10-02, 2025-10-09, 2025-10-30 (whole weeks therefore absent). Affects Asics-, Nike- and Adidas-Centauro series. Monthly aggregates ignore empty weeks rather than treating them as zero; October-2025 averages are based on only 2 weeks (16 + 23).

Product Scope — Direct Channel

Only footwear is tracked. Each brand's API was filtered to exclude apparel, accessories and equipment:

  • Adidas: sport_pt_br facet in the Search API returns footwear-dominant results by default
  • Nike: /calcados/ URL segment restricts to footwear category
  • Asics: allowlist filter on VTEX sport category (Running, SportStyle, Tennis, Skateboarding, Quadra, Trilha, Vôlei, Infantil, Calçados) — excludes Camisetas, Leggings, Meias, Bolsas etc.
  • Under Armour: VTEX category path filter /Gênero/Calçados/Sport/ — the VTEX facet for "Calçados" is non-functional so UA returns all 2 000+ products; footwear is identified post-fetch by path depth

Historical note: before 2026-05-21 the Asics and UA scrapers lacked this filter. W20 (2026-05-17) was the first contaminated week; BQ history was cleaned and JS arrays corrected on 2026-05-21.

Product Scope — Centauro, Olympikus, Mizuno (Aster)

  • Centauro: filtered to grandparent_group = 'Calçados'. Apparel, accessories, socks and equipment are excluded. EAN-level dedup keeps each product 1× (handles marketplace seller duplicates: Centauro-direct preferred, else lowest-price marketplace listing).
  • Olympikus: filtered to subcategory_name IN ('Calçados', 'Tênis'). Model-level dedup by grandparent_id.
  • Mizuno: filtered to subcategory_name = 'Calçados'. Model-level dedup by grandparent_id.

Historical note: before 2026-05-22 the Centauro queries lacked the footwear filter and were mixing footwear with apparel/accessories. Impact on the last closed week was up to +6.8 pp on Adidas % SKUs discounted (37.3% → 44.0% footwear-only). All historical Centauro weeks were re-run with the new filter on 2026-05-22; for 2022 weeks the source's child_is_available field was not yet populated, so the regen accepted NULL as "available" — a slightly looser definition than 2025+ data.

Dedup Granularity by Channel

  • Direct (Adidas/Nike/Asics/UA): model-level (grandparent_id)
  • Netshoes: model-level (product_code)
  • Olympikus / Mizuno: model-level (grandparent_id)
  • Centauro: EAN-level (child_ean) — necessary to dedup the same physical product listed by multiple marketplace sellers. Slightly more granular than other channels (sneakers with many sizes/colors weight more in the average), but a unit-comparable definition would require an additional model-level rollup the source does not currently support cleanly.

Verified 2026-05-22 that Olympikus and Mizuno give the same discount to all sizes of the same model 99.77% of the time, so MAX(child_pct_discount) per model is equivalent to MIN or AVG — no bias introduced by aggregating sizes into a model identifier.
